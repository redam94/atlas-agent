Adaptive Task & Learning Assistant System

A self-hosted, AI-native operating system for a consulting practice — a graph-structured memory, a multi-model routing layer, and a voice-first interface, drawn into one cartography of work. — Vision · April 2026

Generic AI chat treats every question as if it begins from nothing. ATLAS begins from everywhere you've already been — every PDF, repo, email, alert, and prior decision — and routes the right model to the right task, privately, by default.

§ 01 Design Principles

Six commitments that shape every choice.

P · 01

Model Agnosticism

Local Gemma to cloud Claude — a single async streaming interface routes by cost, capability, and privacy without rewriting prompts.

P · 02

Knowledge as Graph

Documents, chunks, entities, and projects are nodes. Retrieval is a personalized random walk — context, not just similarity.

P · 03

Streaming First

Tokens land in the UI as they arrive. WebSockets, async generators, and interruptible voice loops — never wait for completions.

P · 04

Plugin-Oriented

Gmail, GitHub, GCP, Calendar — every integration is a uniform tool the agent can invoke, with schemas auto-derived from Pydantic.

P · 05

Privacy by Default

Sensitive projects are local-only. Cloud LLM calls are explicit, opt-in, and audited per turn — never the silent default.

P · 06

Async Everything

Embeddings, retrieval, tool calls, ingestion — every I/O path is non-blocking. The system breathes while it thinks.

§ 04 Capabilities

What ATLAS does for the practice that nothing else does together.

i

Multi-project workspace

Each engagement has its own RAG scope, model preferences, plugin set, and privacy posture. No bleed between clients.

ii

Graph RAG retrieval

BM25 + vector seed → graph expansion → cross-encoder rerank. Retrieves neighborhoods, not just nearest neighbors.

iii

Multi-model routing

Local Gemma for fast lookups, Claude for code review, Gemini for long-context — selected per request, audited per dollar.

iv

Voice-first agent

Wake-word in the browser, faster-whisper STT, Kokoro TTS, interruptible mid-stream — a real assistant, not a transcript box.

v

Code intelligence

AST-parsed Python, Jupyter cells with markdown context, cross-repo pattern clustering — every prior repo is a knowledge source.

vi

Plugin tool surface

Gmail, GitHub, GCP, Calendar, Notion, Web — uniform tool schemas any model can invoke, with confirmation gates on send.

vii

Sandboxed execution

The agent writes Python, runs it in an isolated sandbox, reads dataframes and plots back into the conversation.

viii

Discord co-presence

Slash commands, daily digests, GCP alert routing — ATLAS available wherever the work already happens.
